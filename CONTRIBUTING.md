# Contributing to Recapstor

Thank you for considering contributing to the Recapstor project! Your contributions are greatly appreciated. Here are some guidelines to help you get started:

## How to Contribute
1. **Fork the Repository**: Click on the "Fork" button on the top right of the repository page to create your own copy.
2. **Create a Branch**: Create a new branch for your feature or bug fix. It’s a good practice to name your branch descriptively.
   
   ```bash
   git checkout -b my-feature-branch
   ```

3. **Make Changes**: Make your changes in the newly created branch.

4. **Commit Changes**: Write clear and concise commit messages describing your changes.
   
   ```bash
   git commit -m "Add my new feature"
   ```

5. **Push Changes**: Push your changes to your forked repository.
   
   ```bash
   git push origin my-feature-branch
   ```

## Submitting a Pull Request
1. Navigate to the original repository where you want to propose your changes.
2. Click on the "New Pull Request" button.
3. Select your branch with the changes and submit the pull request.

## Code of Conduct
Please follow our [Code of Conduct](CODE_OF_CONDUCT.md) in all your interactions with the project.

## Issues
If you find a bug or have a feature request, please create an Issue in the repository. Make sure to provide as much information as possible.

Thank you for your contributions!
